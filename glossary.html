Fit: Building a model

GridSearchCV: a form of model building when an entire search-space of hyper-parameters is defined and each parameter combination is exhaustively tried.  This takes more time and computing power as compared to RandomizedSearchCV.  A specific set of paramter values are supplied in this case.

Hyper-Parameter: Tunable settings that can be adjusted/fine-tuned to make an algorithm/model fit data better.

OverFitting: When the score being measured performs perfectly or almost perfectly on training data with a big discrepancy as compared to test data.  There may be high errors in test data as well.

SMOTE

SUM of Square Errors: This should be minimized in a linear regression model.  This equates to the difference between the predicted Y values and the actual Y values using each X variable coefficient and then the whole value is squared.  The lower the value, the better.  This is a loss-function.

Test-Data: This would be the unseen data or production data.  Never run fit or transform on this set.

Train-Data: This would be the data that is used to train a model.  It is ok to run fit and/or transform here.

Transform: Changing data.  Typically used when standardization, or normalization of data is used.  This is often run in tandem with fit.

RandomizedSearchCV: a form of model building when an entire search-space of hyper-parameters is defined but only a random subset of these combinations are tried in an attempt to find the best model.  This is quicker and can get very close to optimization that is found with GridSearch.  A range of parameter values are defined in this case.

Recall: gives the ratio of true positives to actual positives.  This means low false negatives (IE: lower changes of predicting someone bad as good).

UnderFitting: When the score being measured is low on train and test data.  The model is not able to predict well.
